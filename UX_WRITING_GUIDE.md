# 유심사 UX 라이팅 가이드

> 유심사는 148개국 이상을 지원하는 글로벌 eSIM/USIM 해외 데이터 로밍 플랫폼입니다.
> 서비스 전반에서 사용하는 UX 문구의 원칙, 톤, 카테고리별 예시를 정리한 문서입니다.

---

## 1. 브랜드 보이스 & 톤

| 항목 | 내용 |
|------|------|
| **브랜드 성격** | 친근하고 신뢰감 있는 통신 전문가 |
| **핵심 키워드** | 명확 · 간결 · 친근 · 신뢰 · 전문 |
| **기본 말투** | 해요체 (존댓말) |

### 톤 유형별 가이드

| 톤 | 설명 | 문장 끝 패턴 | 이모지 |
|----|------|-------------|--------|
| 친근 | 일상적이고 따뜻한 말투 | ~요, ~세요 | O |
| 안내 | 명확하고 차분한 정보 전달 | ~해 주세요, ~돼요 | X |
| 축하 | 긍정적이고 밝은 피드백 | ~했어요!, ~완료! | O |
| 주의 | 중요 사항 강조 (불안하지 않게) | ~하시겠어요?, ~사라져요 | X |
| 사과 | 진심 어린 사과 + 해결 의지 | ~죄송해요, ~할게요 | X |
| 긴급 | 시간 제한 강조 (과하지 않게) | ~마세요, ~지금 | O |
| 확인 | 행동 결과를 간결하게 확인 | ~했어요, ~됐어요 | X |
| 설명 | 쉽고 구체적인 정보 제공 | ~할 수 있어요, ~이에요 | X |
| 추천 | 자연스러운 추천 + 근거 제시 | ~이에요, ~해보세요 | X |

## 2. 라이팅 원칙

### 2.1 한 문장은 40자 이내
```
(O) 유심이 3일 내 배송돼요
(X) 주문하신 유심 상품의 배송이 영업일 기준 약 3일 이내에 진행될 예정입니다
```

### 2.2 능동태 사용
```
(O) 결제를 완료했어요
(X) 결제가 처리되었습니다
```

### 2.3 부정문보다 긍정문
```
(O) 정보를 입력하면 바로 진행돼요
(X) 입력하지 않으면 진행할 수 없습니다
```

### 2.4 CTA 버튼은 동사로 시작
```
(O) 유심 주문하기 / 배송 조회하기
(X) 주문 / 조회
```

### 2.5 에러 메시지 = 원인 + 해결방법
```
(O) 카드 정보를 확인해 주세요. 카드번호 16자리를 다시 입력해 주세요.
(X) 결제 오류가 발생했습니다.
```

## 4. 새로운 문구 등록 템플릿

새 UX 문구를 추가할 때 아래 형식을 따라주세요:

```
ID:         [카테고리 약어]-[번호] (예: ord-008)
카테고리:    [카테고리명]
상황:        [어떤 상황에서 노출되는지]
문구:        [실제 표시될 텍스트]
톤:          [친근/안내/축하/주의/사과/긴급/확인/설명/추천]
컴포넌트:    [토스트/모달/배너/페이지/알림/폼 등]
비고:        [추가 설명이 있다면]
```

## 5. 슬랙 봇 사용법

슬랙에서 아래 명령어로 UX 라이팅 가이드를 활용할 수 있어요.

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `/ux 검색 [키워드]` | 키워드로 UX 문구 검색 | `/ux 검색 결제` |
| `/ux 카테고리 [이름]` | 카테고리별 문구 조회 | `/ux 카테고리 주문` |
| `/ux 추천 [상황설명]` | AI가 상황에 맞는 문구 제안 | `/ux 추천 사용자가 잘못된 이메일 입력` |
| `/ux 피드백 [문구]` | 기존 문구의 개선점 분석 | `/ux 피드백 오류가 발생했습니다` |
| `/ux 등록 [카테고리\|문구\|톤\|컴포넌트]` | 새 문구 등록 | `/ux 등록 주문\|주문 완료!\|축하\|토스트` |
| `/ux 톤 [톤이름]` | 톤 가이드 조회 | `/ux 톤 친근` |
| `/ux 원칙` | 라이팅 원칙 조회 | `/ux 원칙` |
| `/ux 도움말` | 전체 명령어 안내 | `/ux 도움말` |
